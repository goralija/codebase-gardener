# Roadmap

> Status: Ground truth
> Purpose: Define product phases.

Detailed epics and tasks live in `docs/15-epics-and-tasks.md`.

## MVP / V1

- Shared JSON contracts and three-lane implementation scaffold.
- Django/DRF/PostgreSQL backend, Redis/Celery workers, React/Vite dashboard, and Python analysis package.
- GitHub App installation and repository selection.
- Gardener-hosted workers.
- Repowise fork integrated as repository intelligence foundation.
- Monorepo support with logical systems.
- Source-truth discovery.
- `GARDENER.md` Repository Constitution.
- Constitution Completeness Score and onboarding questions.
- Repository Entropy Score at repo, system, module, and file level.
- Entropy trend and forecast.
- Detailed first report with architecture violations and maintenance opportunities.
- Manual, scheduled, after N commits, risky module change, PR opened, and CI failure triggers.
- Multiple focused non-conflicting maintenance PRs.
- Conservative autonomous PR safety policy.
- `.gardener/profile.yaml` learning memory.
- Pricing foundation: base per managed repository times complexity multiplier.

## V1 hardening

- Stronger PR outcome learning.
- Better forecast quality.
- Stronger monorepo logical-system detection.
- Improved ROI estimates.
- More parser and ecosystem coverage.
- Better constitution question UX.
- Improved worker isolation and scale.

## Later product expansion

- Full security-platform integrations.
- GitLab and Bitbucket support.
- Slack/Jira notification and workflow integrations.
- Customer-hosted/local hosted deployment for large customers.
- Explicit auto-merge policies after high customer trust and stronger safety gates.
- Organization-wide multi-repo intelligence.

## Deferred

- Broad enterprise compliance before product-market signal.
- Autonomous large refactors.
- Direct risky business-logic changes.
- Dashboard-only positioning.
